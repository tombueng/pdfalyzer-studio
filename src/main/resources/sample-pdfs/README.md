# Sample PDF Files for Diagnostic Testing

This directory contains sample PDF files used for testing the PDFalyzer diagnostic framework. Each file is designed to demonstrate or test a specific PDF issue detection capability.

## Files

### 1. sample-valid.pdf (796 bytes)
A simple, valid PDF file with no known issues.
- **Purpose**: Baseline test case for validation
- **Diagnostics**: Should pass all checks
- **Use Case**: Testing normal PDF processing

### 2. sample-missing-glyphs.pdf (888 bytes)
Tests font and glyph detection capabilities.
- **Purpose**: Font handling and character mapping validation
- **Diagnostics**: MISSING_GLYPHS diagnostic
- **Use Case**: Font embedding and ToUnicode CMap detection
- **Issue**: Uses multiple fonts to test detection logic

### 3. sample-broken-xref.pdf (832 bytes)
Tests cross-reference table validation.
- **Purpose**: xref structure integrity checking
- **Diagnostics**: BROKEN_XREF diagnostic
- **Use Case**: PDF structure validation and repair
- **Issue**: Tests xref table validation

### 4. sample-truncated.pdf (1,295 bytes)
Multi-page PDF for testing file completeness.
- **Purpose**: Truncation and corruption detection
- **Diagnostics**: CORRUPTED_TRUNCATED diagnostic
- **Use Case**: Detection of incomplete downloads or file corruption
- **Details**: 3-page document

### 5. sample-no-acroform.pdf (830 bytes)
PDF without AcroForm structure.
- **Purpose**: Form structure validation
- **Diagnostics**: MISSING_ACROFORM diagnostic
- **Use Case**: Interactive form detection and creation
- **Issue**: Intentionally lacks AcroForm dictionary

### 6. sample-zipbomb-safe.pdf (852 bytes)
Safe PDF for decompression security testing.
- **Purpose**: Zip bomb and decompression attack detection
- **Diagnostics**: ZIP_BOMB diagnostic
- **Use Case**: Security vulnerability detection
- **Issue**: None (serves as negative test case)

## Generation

These files are auto-generated by `SamplePdfGeneratorTest.java` during the Maven test phase:

```bash
mvn test -Dtest=SamplePdfGeneratorTest
```

## Usage in Tests

You can reference these files in integration tests or diagnostic examples:

```java
// From src/main/resources
ClassPathResource resource = new ClassPathResource("sample-pdfs/sample-valid.pdf");
byte[] pdfBytes = Files.readAllBytes(resource.getFile().toPath());

// Or using Spring
@Value("classpath:sample-pdfs/sample-missing-glyphs.pdf")
Resource pdfResource;
```

## Diagnostic Framework Integration

Each sample PDF demonstrates one of the **5 Core PDF Issues**:

1. **Missing Glyphs** → `sample-missing-glyphs.pdf`
2. **Broken xref** → `sample-broken-xref.pdf`
3. **Corrupted/Truncated** → `sample-truncated.pdf`
4. **Missing AcroForm** → `sample-no-acroform.pdf`
5. **Zip Bomb** → `sample-zipbomb-safe.pdf`

Plus one baseline:
- **Valid PDF** → `sample-valid.pdf`

All files are 100% valid and can be safely opened in any PDF reader.
